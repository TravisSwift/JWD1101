Using Multiple Classes

We can assign multiple classes to a single element. It will have all of the properties of each class. We can use multiple classes to make highly customized elements without re-writing the same CSS code over again. Using multiple classes can make our CSS very flexible and easy to understand.

In the case where properties of one class conflict with properties of another class, the class that appears later in the CSS file will win. The exception to this is when a class using the !important rule. Additionally, class rules will always lose to ID rules.

    Reasons for using mutliple classes

  • Reuseability
  • Customization
  • Classes names can be more meaningful
  • Helps create a consistent look and feel

    Reasons to avoid multiple classes

  • Conflicts between class properties can be difficult to troubleshoot
  • Requires careful ordering of the Style Sheet
  • Requires more planning
  • HTML becomes "heavy" to read
