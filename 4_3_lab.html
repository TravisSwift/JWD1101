HTML ELEMENTS

An HTML document is structured in a hierarchical manner, consisting of elements nested within other elements. HTML structure is designed to make coding and reading of websites understandable and predictable. On a whole page level, the html element is the root element and it defines the entire HTML document. It has a start tag html and an end tag /html. Inside the html element, there is a body element, and the rest of the page's elements and content are written inside the body element.

Understandable

The standards of HTML structure guide us to create code that is easy to understand. Using semantic elements gives us easily identifiable elements, which is important not just for the developer but also for clients or other developers who review the code. This is also helpful when collaborating on projects so developers can agree on the basic and minimum standards of coding. Elements are not just divs; paragraphs p and heading tags h1, h2, etc. are also elements and can be manipulated in the same way.

It is important that HTML structure be easily understood by both a coder and a web browser. It is also crucial for screen readers and accessibility. The use of the figure element allows the browser to know there is an image in the element, and figcaption gives an on-screen text description while the alt attribute provides additional image information.

Predictable

The use of divs and semantic elements div, article, main, header, section, etc. puts content in blocks that have predictable results with alignment and placement on the page. Understanding HTML structure and the use of these elements allows developers to use element names, class names, and/or IDs to predictably manipulate the layout of elements and when adding cascading style sheets (CSS) to style the page.

The use of divs and semantic elements follows standards but is also interpreted in a common way that allows creativity and personalization by the developer. One person might prefer to use very creative element, class, and ID names, while another uses simple and repetitive names, and both approaches will behave the same way on the web page.

Summary

While analyzing the three pages, the nav, header, and footer elements are easily identified. The divs, sections, and main elements are easy to see, but the names of those elements can vary based on the style of the developer.
On Weird Al's website, the header is used as a full-page image with content below. The header could have been a small image like a banner, but it would not have the same impact as Al's face on the entire page. The section element could have been named main, or simply div. The article elements could have been named figure, img, div, and they would have behaved the same. The div element containing the call to action could also have been named div or article, or any other name the developer preferred.

The YouTube page uses very simple naming conventions and is a great example of stacked divs and a sidebar layout.

The Homestar Runner web page is a simplistic layout using very few elements. The main page uses a header element as one large image and link with divs containing links to specific content and a simple footer.

Using the same standards of HTML structure, we have three pages using elements in the same way that produce different results because of the CSS applied. Regardless of how the page looks, they follow the standards, naming conventions, and use of elements as any other modern website. Standards and common practices make coding accessable and understandable to all, new coders, experienced developers, web browsers and screen readers that create a consistent, understandable, predictable result.
